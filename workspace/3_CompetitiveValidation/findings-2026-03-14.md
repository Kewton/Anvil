# Competitive Validation Findings 2026-03-14

## Scope

This review compares the current Anvil implementation against `vibe-local` on the Phase 9 checklist.

Sources used:

- measured `--help` startup timings on 2026-03-14
- `workspace/1_ProductDefinition/vibe-local-analysis.md`
- `workspace/1_ProductDefinition/anvil-comparison-axes.md`
- current Anvil code and tests
- `vibe-local` README and CLI surface

## Results by Checklist Item

### 1. Startup and first-use flow against `vibe-local`

Result:

- `vibe-local` wins first-use packaging.
- Anvil is close on raw startup, but still source-tree oriented.

Evidence:

- `vibe-local` publishes install scripts and a direct `vibe-local` command.
- Anvil currently expects local source checkout plus `cargo run`.
- `--help` startup timing:
  - Anvil: 0.21 / 0.22 / 0.22 sec
  - vibe-local: 0.19 / 0.19 / 0.19 sec

### 2. Iteration latency against `vibe-local`

Result:

- provisional edge to Anvil in current implementation simplicity.

Reasoning:

- Anvil has a leaner current runtime path, typed provider normalization, and a smaller feature surface.
- `vibe-local` carries much richer mature behavior, but that also means more orchestration in its main runtime.

Confidence:

- moderate

### 3. Interruption and recovery behavior against `vibe-local`

Result:

- `vibe-local` still leads overall.

Reasoning:

- `vibe-local` has proven interrupt/session/rollback behavior and broader test maturity.
- Anvil has explicit state transitions, persisted pending approvals, and session normalization, but less operational history.

### 4. Long-session context handling against `vibe-local`

Result:

- `vibe-local` leads clearly.

Reasoning:

- `vibe-local` already exposes compaction, RAG, session management, and mature resume tooling.
- Anvil currently has context visibility and resume, but not compaction quality or large-repo retrieval yet.

### 5. Remaining gaps vs Claude Code-level UX

Current gaps:

- packaging and install ergonomics
- richer slash commands and extension lifecycle
- end-to-end typed tool execution wired into runtime/provider flow
- long-session compaction and retrieval
- more mature approval/recovery guidance

### 6. Current Anvil advantages

- clearer operator-console identity
- stronger module boundaries
- easier path to extensible slash commands and typed tools
- better immediate visual distinction between user, agent, and tool output

## Overall Conclusion

As of 2026-03-14:

- Anvil already beats `vibe-local` on console clarity and architectural extensibility.
- `vibe-local` still beats Anvil on packaging, long-session maturity, and operational hardening.
- The next highest-leverage work for Anvil is not more CLI polish; it is long-session capability and typed tool/runtime integration.
