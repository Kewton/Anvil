# Parallel Issue Orchestration - 方法論ドキュメント

developブランチをオーケストレーターとして、複数Issueを並列開発するための方法論。
**開発 → PR → マージ → UAT → 修正ループ → 完了** の全ライフサイクルをカバー。

## 全体フロー

```
Phase 1  準備        worktree作成（並列）
  │
Phase 2  開発        /pm-auto-issue2dev（並列）
  │
Phase 3  品質確認    cargo clippy / test（並列）
  │
Phase 4  PR作成      /create-pr（並列）
  │
Phase 5  マージ      develop に順次マージ（直列）
  │
Phase 6  UAT         /uat（developで実行）
  │
Phase 7  判定 ─── 全PASS ──→ Phase 8 完了
  │
  └── FAIL ──→ featureで修正 → 再PR → 再マージ → Phase 6 に戻る
```

## ドキュメント構成

| ファイル | 内容 |

⏺ 3つのスラッシュコマンドを登録しました。

  ┌────────────────────┬───────────────────────────────────────┬───────────────────────────────────────────────────────────────────────────────┐
  │      コマンド      │               ファイル                │                                     概要                                      │
  ├────────────────────┼───────────────────────────────────────┼───────────────────────────────────────────────────────────────────────────────┤
  │ /orchestrate       │ .claude/commands/orchestrate.md       │ 全ライフサイクル統括（準備→開発→設計突合→品質→PR→マージ→UAT→修正ループ→完了） │
  ├────────────────────┼───────────────────────────────────────┼───────────────────────────────────────────────────────────────────────────────┤
  │ /pr-merge-pipeline │ .claude/commands/pr-merge-pipeline.md │ PR作成→CI通過待ち→マージ順序決定→順次マージ→統合検証                          │
  ├────────────────────┼───────────────────────────────────────┼───────────────────────────────────────────────────────────────────────────────┤
  │ /uat-fix-loop      │ .claude/commands/uat-fix-loop.md      │ FAIL分析→ワーカー修正→再PR→再マージ→再UAT（最大N回ループ）                    │
  └────────────────────┴───────────────────────────────────────┴───────────────────────────────────────────────────────────────────────────────┘

  コマンド間の連携:

  /orchestrate 24 25 --full
      ├── Phase 3: 並列開発（/pm-auto-issue2dev を各ワーカーに send）
      ├── Phase 6: /pr-merge-pipeline 24 25
      ├── Phase 7: /uat 24 25
      └── Phase 7: /uat-fix-loop（FAIL時）

  個別にも使えます:
  - /pr-merge-pipeline 24 25 - 開発完了後、PRからマージまで
  - /uat-fix-loop 24 - UAT失敗後、修正ループだけ
