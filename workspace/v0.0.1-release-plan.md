# Anvil v0.0.1 リリース作業計画

## 現状サマリー

| 項目 | 状態 |
|------|------|
| ブランチ | main（origin/mainに対し104コミット先行） |
| テスト | 108件全パス |
| clippy | 警告0件 |
| cargo fmt | **71ファイルに差分あり（要修正）** |
| タグ | なし |
| Cargo.toml version | 0.1.0（**0.0.1に変更必要**） |
| CI | ci.yml あり（fmt, clippy, test, build） |

---

## 必須作業（リリースブロッカー）

### 1. cargo fmt の適用
- **理由**: CIで `cargo fmt --check` が失敗する（71ファイル）
- **作業**: `cargo fmt` を実行してコミット
- **見積**: 5分

### 2. Cargo.toml のバージョン修正
- **理由**: 現在 `version = "0.1.0"` だが、初回リリースは v0.0.1
- **作業**: `version = "0.0.1"` に変更してコミット
- **見積**: 1分

### 3. CI ワークフローの統一
- **理由**: `ci.yml` と `ci-pr.yml` が重複。ci.yml は main push + PR 向け、ci-pr.yml は新規追加分
- **作業**: ci-pr.yml の内容を ci.yml に統合し、ci-pr.yml を削除
- **見積**: 10分

### 4. origin/main へのプッシュ
- **理由**: 104コミットがローカルのみ。CI実行にはリモートへのプッシュが必要
- **作業**: `git push origin main`
- **見積**: 1分

### 5. CI パス確認
- **理由**: リモートでCIが通ることを確認する
- **作業**: `gh run list` で確認。失敗時は修正
- **見積**: 5-15分

### 6. Git タグ作成とGitHub Release
- **理由**: v0.0.1 としてリリースを公開する
- **作業**:
  ```bash
  git tag -a v0.0.1 -m "v0.0.1: initial release"
  git push origin v0.0.1
  gh release create v0.0.1 --title "v0.0.1" --generate-notes
  ```
- **見積**: 5分

## 推奨作業（リリース品質向上）

### 7. CHANGELOG.md 作成
- **理由**: リリースノートの基盤。今後のリリースで差分を追跡できる
- **作業**: 104コミットの要約をCHANGELOG.mdとして作成
- **見積**: 15分

### 8. LICENSE ファイル追加
- **理由**: README.md に「MIT」と記載しているが、LICENSEファイルが存在しない
- **作業**: MIT LICENSEファイルを作成
- **見積**: 2分

### 9. cargo build --release のバイナリサイズ確認
- **理由**: リリースバイナリの基本的な健全性確認
- **作業**: `cargo build --release && ls -lh target/release/anvil`
- **見積**: 2分

### 10. README.md の最終確認
- **理由**: v0.0.1時点の機能と一致しているか確認
- **作業**: README.mdの内容と実際の動作を照合
- **見積**: 10分

## 実行順序

```
Step 1: cargo fmt                          ← フォーマット修正
Step 2: Cargo.toml version → 0.0.1        ← バージョン修正
Step 3: CI統合 (ci-pr.yml → ci.yml)        ← CI整理
Step 4: LICENSE追加                        ← ライセンス
Step 5: CHANGELOG.md作成                   ← リリースノート
Step 6: git push origin main              ← リモート同期
Step 7: CI パス確認                        ← 品質ゲート
Step 8: git tag v0.0.1 + gh release       ← リリース
Step 9: develop ブランチ作成               ← ブランチ戦略
```
